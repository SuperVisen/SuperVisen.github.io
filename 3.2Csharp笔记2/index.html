本篇介绍面向对象的思想，可以理解为把架子先摆好，后续把书放上去。在入门的时候，我曾经说过C#程序主体是类，甚至连程序都是类，这也是面向对象语言的特点，该篇可以在复习中慢慢体会。

何为“面向对象”

    我们生活中经常会遇到很多定义或者概念，我相信大家肯定也会困惑，为什么这样定义或者说出于什么目的去把这类事物描述出来

   比如为什么有加法？这个大家肯定都能解释了，是为了满足生活中计算的需要。加法对应的就是加法的逻辑，你脑袋里第一个思考的肯定是两个数对应的位相加，甚至连计算的方法都想好了。

能不能有一种方法能把这个思考过程表示出来呢？有，面向对象！

  1. 首先我们得有需求，发明东西肯定是为了简便，为了生产。面向对象的需求是，想要描述出我想编程的所有现实事物（以汽车为例）
  2. 那现实事物对应哪些特点？我们大脑认出那是辆汽车，肯定是得有四个轮子，有汽车品牌，可以移动等等。那我们抽象出来完成建模，让我们程序模拟大脑的思考过程。可以得到程序中汽车的模型，以后碰到个东西，程序比对判断是否符合这个模型。符合的话，我们会喊它是一种/类汽车（我们定义的汽车类）—— 定义类：从实际到抽象
  3. 前一步只是抽象出汽车这个概念呀，那怎么描述现实的汽车呢？对啊，那比如我碰到个现实中的比亚迪电动汽车（待描述的对象），程序怎么用刚才定义的汽车类去描述？有没有发现其实这是定义类的逆过程，把抽象转化成现实事物，我们把这过程称为实例化，程序中表示的对象称为实例

思考完后，进行总结：

类（class）是对现实世界事物的模型，将现实事物进行抽象，事物包括“物质”（实体）与“运动”（逻辑）

比如，汽车有四个轮子（实体）和能够运动（逻辑）

对象也叫实例，现实中称为对象，程序世界称为实例，是类经过实例化后得到的内存中的实体

通过类，我们可以创建对象，这就是实例化

面向对象编程也就是针对现实中的具体事物进行编程，包含定义类和实例化的过程

CSharp里的面向对象

理解完面向对象编程，那Csharp被设计如何表现或实现面向对象呢？使用过其他的面向对象语言（C++、Java、Python、Javascript等）的小伙伴，可以发现只是大同小异

定义类

Csharp中采用单词Class的程序块表示某个类以及该类的特征（成员）

我们先介绍类的三大成员，也就是类的特征。我们翻看微软的某个class的时候，都能发现这里介绍的这三大成员占了大半部分（后续成员会在后面进行讲解）

  • 属性Property：存储数据，组合起来表示类或者对象当前的状态。比如car汽车类，有Price价格属性（car.Price）、Brand品牌属性（car.Brand）、Power能源属性（car.Power）等等

  • 方法Method：由C语言的函数function进化而来，表示类或者对象能做什么**（动作）**

  • 事件Event：类或者对象通知其他类或者对象的机制（不同类或对象的通信），为C#所特有（Java是通过其他方法实现），但记住不能滥用

虽然微软文档中类Class基本都有这三大成员，但是某些特殊类或对象在这三个成员的侧重点不同

  • 模型类或对象侧重属性：如Entity Framework
  • 工具类或对象侧重方法：如Math、Console
  • 通知类或对象侧重事件：如各种Timer

这里我们以上述的比亚迪电动汽车（对象）为例，它的特征是

  • 标价20万
  • 品牌是比亚迪
  • 能源是电池
  • 具有四个轮子
  • 能够移动

定义的汽车类，它应该具有特征：价格、品牌、能源、轮子数量、能够移动

静态成员和实例成员

定义类的时候，相信大家应该发现了Move方法前面有个static关键字，加与不加static声明的成员有什么不同呢？

关键字是预定义的保留标识符，作用是让程序员标记和区分不同的使用情况

这里的static用来声明静态（static）成员，而静态成员对应着实例成员。成员不仅包括字段、方法、属性、运算符、事件和构造函数，还包括类class，类中的“成员”可以是类

  • 静态（static）成员，也叫类成员，在语义上表示“类的成员”
  • 实例（非静态）成员，也叫对象成员，在语义上表示“对象的成员”

通俗理解：实例表示的是对象（类实例后）的状态，静态表示的是该类的状态。理解起来就像某班级某学生的学习状态（学习刻苦，态度认真）和该班的学生状态（尖子班，学习氛围浓郁）

那么这两种成员声明方式该如何选择，或者回到那个问题，有什么不同？

不同 静态成员 非静态成员
访问方法 必须由类名调用，不能使用对象调用
静态成员只属于类，访问时需使用“类名.静态成员”		 必须实例化后，再使用“对象.实例成员”进行访问
不同类的成员 静态类不能实例化，只能有静态成员不能有实例成员 非静态类可以有静态成员和非静态成员
类第一次加载时内存分配 该类下的所有的静态成员都会被加载
静态成员只分配一次，有且只有一份存储空间
无论new出多少个实例，它也只是有那一个空间静态成员会被加载到静态存储区，直到程序退出时才会被释放		 实例成员有多少个对象，就会创建多少个对象
每产生一个实例都要重新分配一个存储空间

总结：

  1. 静态成员不能实例化，只能由类名调用——“类名.静态成员”；第一次加载时，会分配一个存储空间，new也是引用这部分空间

  2. 非静态成员必须实例化后，再使用“对象.实例成员”进行访问；有多少个对象，就会创建多少个对象，需注意内存管理
