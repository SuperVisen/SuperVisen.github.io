使用Github和Hexo搭建属于自己的博客

发表于2021-11-04|更新于2022-07-20|操作指南博客相关
|阅读量:

引子

&nbsp;&nbsp;&nbsp;&nbsp;👋这里是Visen，取名原因除了名字谐音，也是因为vise是老虎钳的意思，虎年嘛，钳字又让我觉得能钳制住困难；有时候取名叫supervisen，是因为supervisor是老板的意思，不想当打工人了！最近发现自己能够作为内容输出者，想着搭建独一无二的个人博客。通过博客的形式来驱动自己学习，方便后续回顾，内化知识的同时也从分享中得到快乐。当然也不排斥CSDN、博客园之类，但是用何同学的话来总结就是“不够酷”，看着网上铺天盖地的教程，我知道自己有得忙活。在这里也想记录一下建站的整个过程。

这里只提供Window环境下安装指南。大家也可以通过国内的Gitee来建博客，但是需要手持身份证照片，有点尬，我拒绝了哈哈哈。

博客方案

目前搭建博客有几个常用方案

  • 静态网站生成：Git + Github Pages + Markdown + jekyll/hexo/hugo/手写前端/Gridea（免费）
  • 第三方：CSDN、博客园、简书等（不够酷）
  • 内容管理系统 CMS：虚拟主机（云服务器）+插件+Wordpress/Ghost/手写前后端（付费）

架不住我没钱呀，于是选了第一个😂（哦，表情包可以配置或者直接粘贴：表情包配置方法/表情包网站)。

如果是计算机纯小白，也没有linux、git和前端基础不妨试着使用Gridea，不用在本地搞太多东西，更单纯地写博客，里面还自带表情包，只是界面主题稍微少了一些，个性化也少一些。这里放个Gridea官方教程。

博客环境搭建

1.Git环境

Git （管理代码）是一个开源的分布式版本控制统，用于敏捷高效地处理任何或小或大的项目。Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。与常用的版本控制工具 CVS, Subversion 等不同，它采用了分布式版本库的方式，无需服务器端软件（IIS、Apache、Nginx等）支持。

  1. 下载对应系统的Git（清华镜像），这里选择最新的64位exe安装包，点击安装，一直点next就可以（可更改安装路径）。

  1. 安装完成后，右键就会出现git选项，可以在任何文件夹位置打开界面，命令行使用linux命令。

  1. 初始化git配置，配置个人的用户名称和电子邮件地址。使用 –global 选项，那么更改的是用户主目录配置文件，以后你所有的项目默认使用该用户信息。（Windows用户主目录：C盘/用户/你的用户名）

2.Node环境

Node.js （hexo环境依赖）是一个开源和跨平台的 JavaScript 运行时环境。 具有独特的优势，数百万为浏览器编写 JavaScript 的前端开发者无需学习完全不同的语言，就可以编写除客户端代码之外的服务器端代码。（适用于前后端分离项目，可以代替Python/PHP这种P类等后端语言）

  1. 下载Node.js（清华镜像），点击安装。这里推荐最新版就行，官方推荐是Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本

  1. 安装成功后，进入git bash，查看node和npm版本，搭建成功（tip：git中复制快捷键是Ctrl+Insert，粘贴是Shift+Insert）

3.Hexo环境

hexo是一种基于Node.js的快速、简洁且高效的博客框架，依赖少易于使用，可生成静态网页托管在一些代码仓库github、gitee等。

  1. 创建文件夹（博客本地位置，也是根目录）用来布置hexo，直接在文件夹中右键选择git中用npm安装，等待安装完成。（没反应的话，就重启一下电脑）

  1. 一些显示并不是错误，没有其他问题就是成功了。

  1. 初始化hexo，并且安装好依赖。可以看到hexo已经在文件夹里布置好了。

  1. 通过目前默认主题landscape来本地部署博客，可能会出现Accessing non-existent property的问题，不用管，是stylus模块的问题（问过开发者）

  1. 浏览器访问 http://localhost：4000，看到landscape的界面，本地部署完成，使用Ctrl+C可以关闭服务。

  1. 了解下hexo是怎么放文件的（文件结构），后续方便理解。

  • 执行清理缓存命令，可以回到初始化后的节点，发现少了个public文件夹，该文件夹就是通过执行_config.landscape.yml产生的网页文件（注：此时任何文件夹存在的文件都最好不要删除，不然就等于框架缺了一角，会出现错误）

  • 使用hexo g后，再次生成完整的文件夹。其文件结构如下：（使用Mermaid代码块实现）

      graph LR
A(hexo根目录)-->B1(node_modules//依赖的模块)
A(hexo根目录)-->B2(public//通过yml主题配置文件生成的页面)
A(hexo根目录)-->B3(source//资源文件)
B3-->C1(_posts//存放md文章,不能为空)
A(hexo根目录)-->B4(themes//主题文件夹)
A(hexo根目录)-->B5(_config.yml//hexo博客的设置)

    但是Mermaid写法还是得根据自己主题设置，比如我选择的主题有语法改变

    补充：_config.yml的理解可以参考hexo官方文档

4.Github环境

GitHub是世界上最大的代码托管平台（代码仓库），可以把代码放上面开源，分享自己的代码，也可以随时下载到本地。

  1. 注册Github，并登录。git bash输入ssh-keygen命令，在用户主目录的.ssh文件夹中会生成两个.ssh/id_rsa（私钥文件）和.ssh/id_rsa.pub（公钥文件）。

  1. 新建SSH Key，输入刚复制的公钥，绑定成功。

  2. 测试下连接，显示连接成功！

  3. 进入github网站，点击your repositories—>new来新建仓库。

  4. Github仓库名形如 <用户名>.github.io。开启 Pages 服务后可以直接通过 http://<用户名>.github.io 访问

  1. 在设置中开通github pages服务，任意选择一个主题，点击提交更改。

  1. 在hexo根目录中的**_config.yml**（hexo的配置文件），将deploy部署项改成github以部署到github仓库。

更换主题

虽然这时候已经可以写博客了，但是界面不够酷！那就开始更换主题吧！👀

1.主题挑选

官网里提供蛮多的主题，也可以参考知乎等平台找，修改的方法大相径庭。这里我使用的是Butterfly。确定后，进入GitHub克隆代码到Hexo根目录/themes目录下。

根据主题下面的installation安装备注，可以获取很多信息，首先知道有两类安装方式

总结

&nbsp;&nbsp;&nbsp;&nbsp;目前实现的是本地博客和GitHub Pages绑定，图片上传没有图床的话不够便利，推荐采用PicGo+GitHub做图床。采用使用Typora作为写作工具，操作简单（收费？最终免费版或者使用Markdown Nice可以导出到公众号/知乎，并且支持图床和修改自定义CSS样式）；也可以采用Vscode代替Typora，再加上PicGo插件实现自动上传图片到Github图床。好了，大家可以不断通过hexo g && hexo d上传代码，主题配置提交更改前必须先使用hexo clean。这里再放个Markdown教程(有可能git会在上传时提示“warning: LF will be replaced by CRLF”，附上解决方法)。
